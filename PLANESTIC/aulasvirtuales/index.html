    La solución se implementa con AWS Cloud Development Kit (AWS CDK), que permite a los usuarios definir recursos de aplicaciones en la nube mediante lenguajes de programación familiares. La solución se implementa utilizando dos zonas de disponibilidad (AZ) para proporcionar alta disponibilidad.

    En el punto de enlace de la aplicación, se crea una distribución de Amazon CloudFront y se utiliza como punto de enlace para que los usuarios finales accedan a la aplicación de Moodle. CloudFront mejora el rendimiento de la aplicación al servir el contenido cerca de donde se encuentran los usuarios finales con baja latencia. La solución crea y asocia una lista de control de acceso web (ACL) de AWS Web Application Firewall (AWS WAF) con la distribución de CloudFront con el grupo de reglas administrado de la lista de reputación de IP de Amazon habilitado. También puede habilitar reglas adicionales según sea necesario en esta ACL web. Detrás de CloudFront, el tráfico de la aplicación de Moodle se equilibra con la carga mediante Application Load Balancer (ALB) y se protege con cifrado en tránsito con el certificado TLS almacenado en AWS Certificate Manager (ACM). ALB distribuye automáticamente el tráfico entrante entre varias instancias de Moodle. Supervisa el estado de sus objetivos registrados y enruta el tráfico sólo a los objetivos en buen estado. ALB escala el balanceador de carga a medida que el tráfico entrante cambia con el tiempo.

    El tráfico entrante distribuido por ALB lo recibe un grupo de instancias de Moodle que se ejecuta en Amazon ECS mediante una combinación de Fargate y Fargate Spot. El servicio de Amazon ECS organiza automáticamente varias tareas de Amazon ECS que ejecutan los contenedores de Moodle. La imagen del contenedor para la aplicación se basa en la imagen de Bitnami Moodle Docker con algunas modificaciones para permitir la integración del almacenamiento en caché de Redis. La imagen del contenedor se almacena en Amazon Elastic Container Registry (Amazon ECR), que proporciona un registro de contenedores completamente administrado.

    Para centralizar y compartir archivos moodledata en varias instancias de Moodle, se requiere un sistema de archivos compartidos para esta solución. Amazon Elastic File System (Amazon EFS) es un sistema de archivos elástico simple, sin servidor, que se configura y se olvida. Amazon EFS simplifica la configuración, el escalado y la optimización de costos del almacenamiento de archivos en AWS. Amazon EFS se implementa y monta en las tareas de Amazon ECS para usarse como sistema de archivos subyacente de Moodle y moodledata. La base de datos de Moodle también está centralizada e implementada en una instancia de Amazon Relational Database Service (Amazon RDS). Con Amazon RDS, un servicio administrado, se puede configurar, operar y escalar una base de datos relacional en la nube. Proporciona capacidad rentable y redimensionable, al mismo tiempo que administra tareas de administración de bases de datos que requieren mucho tiempo, lo que le permite concentrarse en sus aplicaciones y negocios.

    Por último, para mejorar el rendimiento general de la aplicación, Moodle tiene un mecanismo de almacenamiento en caché incorporado que utiliza la memoria, el sistema de archivos o el almacenamiento de caché externo como Memcached o Redis. Esta solución utiliza Amazon ElastiCache para Redis como un almacén de caché centralizado. Amazon ElastiCache para Redis simplifica la implementación y la ejecución de nodos de servidor compatibles con el protocolo Redis en AWS.
