    Oficina De Investigación ODI

    OMP (Open Monograph Press)

    El montaje de Open Journal Systems (OJS) y Open Monograph Press (OMP) en un entorno de nube con Amazon Web Services (AWS) puede ser una solución efectiva para gestionar publicaciones académicas y sistemas de gestión de revistas. A continuación, se detalla el proceso para desplegar estos sistemas utilizando instancias EC2, Amazon RDS, un balanceador de carga y AWS WAF.

    1. Preparación y Planificación

    Desplegar OJS y OMP de manera eficiente en AWS, garantizando la disponibilidad, escalabilidad y seguridad.

    Componentes Principales: * Instancia EC2: Para alojar las aplicaciones OJS y OMP. * Amazon RDS: Para la base de datos. * Balanceador de Carga: Para distribuir el tráfico entre las instancias EC2. * AWS WAF (Web Application Firewall): Para proteger las aplicaciones contra amenazas web.

    1. Configuración de la Infraestructura

      a. Creación de Instancias EC2

      • Amazon EC2 (Elastic Compute Cloud) proporciona servidores virtuales escalables para ejecutar aplicaciones.
      • Seleccionar una AMI (Amazon Machine Image):
      • Elegir una AMI adecuada para el sistema operativo requerido (por ejemplo, Amazon Linux 2, Ubuntu).

      b. Configurar la Instancia:

      • Seleccionar un tipo de instancia adecuado para el rendimiento requerido (por ejemplo, t3.medium o m5.large).
      • Configurar el almacenamiento (EBS) según las necesidades de la aplicación.
      • Configurar el grupo de seguridad para permitir el tráfico HTTP/HTTPS (puertos 80 y 443) y SSH (puerto 22).
      • Instalar el Software Necesario:
        • Apache/Nginx: Servidor web.
        • PHP: Para ejecutar OJS y OMP.
        • MySQL/MariaDB: Aunque se usará Amazon RDS para la base de datos, la instancia EC2 debe tener los clientes necesarios para interactuar con la base de datos.
        • Desplegar OJS y OMP:
        • Descargar y configurar OJS y OMP en las instancias EC2.
        • Configurar los archivos de configuración de OJS y OMP para conectar con la base de datos en Amazon RDS.
        • Configurar el servidor web para servir las aplicaciones correctamente.

      c. Configuración de Amazon RDS

      • Amazon RDS (Relational Database Service) proporciona bases de datos gestionadas.
      • Crear una Instancia RDS:
        • Seleccionar el motor de base de datos compatible (MySQL o MariaDB).
        • Configurar la instancia (tipo, almacenamiento, etc.) según las necesidades de la aplicación.
        • Configurar parámetros de la base de datos y seguridad (grupo de seguridad para permitir el acceso desde las instancias EC2).
        • Migrar los Datos:
        • Si es necesario, importar datos existentes a la base de datos RDS.

      d. Configuración del Balanceador de Carga

      • Elastic Load Balancer (ELB) distribuye el tráfico entrante entre varias instancias EC2 para asegurar alta disponibilidad y escalabilidad.
      • Crear un Load Balancer:
        • Seleccionar el tipo de balanceador de carga (Application Load Balancer (ALB) es adecuado para HTTP/HTTPS).
        • Configurar las zonas de disponibilidad y los listeners (puertos 80 para HTTP y 443 para HTTPS).
        • Configurar el Target Group:
        • Definir el grupo de destino que incluye las instancias EC2 que ejecutan OJS y OMP.
        • Configurar las reglas de enrutamiento y verificar el estado de las instancias.
        • Actualizar la Configuración de DNS:
        • Configurar el nombre de dominio para apuntar al balanceador de carga.

      e. Configuración de AWS WAF

      • AWS WAF (Web Application Firewall) protege las aplicaciones web contra ataques comunes.
      • Crear una Web ACL (Access Control List):
        • Configurar reglas de firewall para bloquear o permitir tráfico basado en IP, geolocalización, patrones de solicitudes y otros criterios.
        • Asociar la Web ACL con el Balanceador de Carga:
        • Aplicar la Web ACL al balanceador de carga para proteger las instancias EC2 que ejecutan OJS y OMP.
        • Monitorear y Ajustar las Reglas:
        • Revisar los registros y métricas de AWS WAF para ajustar las reglas y mejorar la protección.

    2. Monitoreo

      a. Monitoreo: * Utilizar Amazon CloudWatch para supervisar el rendimiento de las instancias EC2, la base de datos RDS y el balanceador de carga. * Configurar alertas para notificar sobre problemas de rendimiento o disponibilidad.

    Desplegar OJS y OMP utilizando instancias EC2, Amazon RDS, un balanceador de carga y AWS WAF en AWS proporciona una solución robusta y escalable para la gestión de publicaciones académicas. Esta configuración garantiza alta disponibilidad, rendimiento escalable y protección contra amenazas web, asegurando una experiencia óptima para los usuarios y la integridad de los datos.
