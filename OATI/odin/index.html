    ODIN (Server) COMPLETADO

    El despliegue del aplicativo ODIN, diseñado para la creación de APIs de consulta en la Universidad Distrital Francisco José de Caldas, se realiza mediante una solución de integración y despliegue continuo (CI/CD) en Amazon Web Services (AWS). Esta solución utiliza GitHub como repositorio de código, AWS CodePipeline para la automatización del flujo de trabajo, y AWS Elastic Beanstalk para la gestión del entorno de ejecución, junto con instancias EC2 para la ejecución del servidor y un balanceador de carga para asegurar la alta disponibilidad y el rendimiento óptimo. El proceso comienza con el código fuente del aplicativo ODIN, desarrollado en Python utilizando el framework Django, que se almacena y versiona en un repositorio de GitHub. Cada vez que se realiza un commit o se actualiza el código en GitHub, se activa el pipeline en AWS CodePipeline. Esta herramienta coordina las etapas del CI/CD, gestionando automáticamente la construcción, pruebas y despliegue del código.

    Durante la fase de integración, AWS CodeBuild se encarga de compilar el código, realizar pruebas automatizadas y generar los artefactos necesarios para el despliegue. Una vez que el código ha pasado las pruebas, AWS CodePipeline inicia el proceso de despliegue hacia AWS Elastic Beanstalk. Elastic Beanstalk proporciona una plataforma gestionada para desplegar y administrar aplicaciones web desarrolladas en Django, simplificando la configuración y el manejo del entorno de ejecución. En el entorno de Elastic Beanstalk, se utilizan instancias EC2 para alojar el aplicativo ODIN. Elastic Beanstalk administra estas instancias, facilitando la escalabilidad automática y el balanceo de carga. El balanceador de carga distribuye el tráfico entrante entre las instancias EC2, asegurando que las solicitudes sean manejadas de manera eficiente y que la aplicación mantenga un alto nivel de rendimiento y disponibilidad.

    Para gestionar configuraciones específicas y credenciales de la aplicación, se configuran variables de entorno en Elastic Beanstalk. Estas variables permiten ajustar parámetros críticos, como la conexión a bases de datos o credenciales de servicios externos, sin necesidad de modificar el código fuente del aplicativo.

    El almacenamiento de datos y archivos asociados con el sistema ODIN se realiza mediante Amazon S3. Este servicio de almacenamiento de objetos proporciona durabilidad y accesibilidad a largo plazo para cualquier archivo que el sistema necesite almacenar y gestionar.

    En resumen, la solución de CI/CD implementada para el despliegue del aplicativo ODIN en AWS asegura una integración continua y despliegue automatizado, facilitando la administración de la infraestructura y el código, mientras garantiza la disponibilidad, escalabilidad y rendimiento de las APIs de consulta. La combinación de GitHub, AWS CodePipeline, Elastic Beanstalk y otras herramientas de AWS optimiza el ciclo de vida del desarrollo y el mantenimiento del sistema, mejorando la eficiencia operativa y la capacidad de respuesta ante cambios y nuevas demandas.

    ODIN (Docker-Compose - Contenerizado) EN PROCESO

    Evidencia de Funcionamiento

    El despliegue del aplicativo ODIN, un sistema para la creación de APIs de consulta en la Universidad Distrital Francisco José de Caldas, se realiza mediante un enfoque de integración y despliegue continuo (CI/CD) en Amazon Web Services (AWS), utilizando Docker para la contenedorización de la aplicación. En esta arquitectura, GitHub sirve como repositorio de código, AWS CodePipeline gestiona la automatización del flujo de trabajo, y AWS Elastic Beanstalk con Docker se encarga del despliegue y la ejecución del aplicativo, con un balanceador de carga para asegurar la alta disponibilidad y variables de entorno para la configuración específica.

    El proceso de despliegue comienza con el código fuente del aplicativo ODIN, desarrollado en Python con el framework Django, almacenado y versionado en GitHub. Cada vez que se realiza un commit o se actualiza el código, AWS CodePipeline se activa para coordinar el flujo de CI/CD. El pipeline incluye varias etapas, comenzando con la construcción y las pruebas del código a través de AWS CodeBuild, que compila el código y ejecuta pruebas automatizadas para garantizar la calidad del software. Una vez que el código ha sido validado, AWS CodePipeline construye una imagen Docker del aplicativo ODIN. Esta imagen encapsula la aplicación y todas sus dependencias, asegurando que el entorno de ejecución sea consistente y reproducible. La imagen Docker se almacena en Amazon Elastic Container Registry (ECR), un servicio de registro de contenedores gestionado que facilita el almacenamiento y la distribución de imágenes Docker.

    AWS Elastic Beanstalk, configurado para usar Docker, toma la imagen desde Amazon ECR y la despliega en un entorno gestionado. Elastic Beanstalk automatiza la provisión de infraestructura, el escalado y el balanceo de carga. El balanceador de carga distribuido entre las instancias de contenedores asegura que las solicitudes sean gestionadas eficientemente, manteniendo un rendimiento óptimo y alta disponibilidad de la aplicación.

    Para manejar configuraciones específicas, como credenciales de bases de datos o variables de entorno críticas, Elastic Beanstalk permite definir estas variables de entorno de manera segura y accesible. Estas variables se utilizan para configurar el comportamiento del aplicativo sin modificar el código fuente.

    El almacenamiento de datos persistentes asociados con el sistema ODIN se gestiona a través de Amazon S3. Este servicio proporciona almacenamiento de objetos duradero y accesible, adecuado para guardar archivos y datos necesarios para el funcionamiento del sistema.

    En resumen, la arquitectura de despliegue del aplicativo ODIN en AWS, utilizando Docker, asegura una integración continua y despliegue automatizado, proporcionando una solución flexible y eficiente. Al emplear GitHub para el control de versiones, AWS CodePipeline para la automatización del flujo de trabajo, y AWS Elastic Beanstalk para la gestión del entorno de contenedores, se optimiza el ciclo de vida del desarrollo, la escalabilidad y el rendimiento del sistema de APIs de consulta, garantizando una operación fluida y adaptable a las necesidades cambiantes.
