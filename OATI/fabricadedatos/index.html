    Fabrica de Datos (Analítica)

    La fabrica de datos surge a raíz de la necesidad de gestionar adecuadamente la data generada por todos los aplicativo y que a su vez permita tomar decisiones adecuadas frente al mismo.

    En apoyo con AWS se realiza un proyecto de crecimiento asociado el cual permite a través del apoyo del proveedor realizar por medio de unos créditos iniciales el piloto del proyecto. Este piloto se establece con la arquitectura anterior permitiendo integrar la data de un primer acercamiento respecto a deserción estudiantil. Se realizan los procesos correspondientes a la gestión de los recursos solicitados a AWS para el despliegue e implementación de este primer piloto, con el fin de evidenciar los resultados obtenidos con los responsables pertinentes y demás interesados en dicho proceso, para ello se solicitó a través de AWS los siguientes recursos mediante esta calculadora de AWS:

    https://calculator.aws/#/estimate?id=9bae821d9286d6f90fb697acead3292640e8d7a2

    Sin embargo, ellos indicadore requerir mejoras a la misma y nos respondieron con la siguiente calculadora.

    https://calculator.aws/#/estimate?id=4e14e5299748edb47865bb88ee0c0a468f0b8791

    Esta arquitectura planteada consta de unas fases que permitirán la visualización ágil de los resultados a su vez de permitir que los administradores de bases de datos se integren más fácil a la solución y ellos quienes están frente a esta información diariamente sean los que extraigan la data requerida para tal finalidad. Consta de las siguientes 4 fases:
    1. Data ingestión (AWS DMS): La ingesta de datos es el proceso de obtener datos de diversas fuentes y llevarlos a un sistema de almacenamiento o procesamiento, como un data warehouse o un data lake. Este proceso es crucial para la analítica y la toma de decisiones, ya que permite consolidar información dispersa en un único lugar. La ingesta de datos puede realizarse de diferentes maneras:
      • Batch Ingestion: Los datos se recopilan y procesan en lotes a intervalos programados. Este método es eficiente para grandes volúmenes de datos que no requieren actualización en tiempo real.
      • Real-Time Ingestion: Los datos se procesan en tiempo real o casi en tiempo real, lo que permite obtener información actualizada de manera inmediata. Este enfoque es esencial para aplicaciones que requieren respuestas rápidas.
      • Streaming Ingestion: Similar a la ingesta en tiempo real, pero se centra en el flujo continuo de datos, permitiendo la captura constante de información desde diversas fuentes.
    2. Scalable Supply Chain Data Lake: Es un repositorio centralizado y flexible diseñado para almacenar grandes volúmenes de datos relacionados con la cadena de suministro de manera escalable. Este enfoque permite a las organizaciones recopilar, procesar y analizar datos provenientes de diversas fuentes, como proveedores, sistemas de gestión de inventarios, logística y ventas. Tiene como características clave:
      • Escalabilidad: Permite el crecimiento de datos sin limitaciones, adaptándose a las necesidades cambiantes del negocio.
      • Variedad de Datos: Soporta múltiples formatos de datos (estructurados, semiestructurados y no estructurados), lo que facilita la integración de información de diferentes sistemas.
      • Accesibilidad: Los datos pueden ser fácilmente accesibles para diferentes usuarios y herramientas analíticas, lo que favorece la colaboración y la toma de decisiones informadas.
      • Análisis Avanzado: Permite la aplicación de técnicas de análisis de datos, como el machine learning y la analítica predictiva, para optimizar la cadena de suministro, mejorar la previsión de demanda y reducir costos.
      • Agilidad: Facilita la rápida incorporación de nuevos datos y la adaptación a cambios en el entorno del mercado.
    3. Seamless Data Movement: Se refiere a la transferencia fluida y continua de datos entre diferentes sistemas, aplicaciones y entornos, sin interrupciones ni complicaciones. Este concepto es esencial para garantizar que la información esté siempre disponible y actualizada, facilitando así la toma de decisiones y la colaboración en tiempo real. En el contexto de AWS implica la transferencia eficiente y continua de datos utilizando herramientas como AWS Glue, Data Catalog y Crawler. Estas herramientas permiten integrar, organizar y transformar datos de manera fluida en un entorno en la nube.
      • AWS Glue: Es un servicio de ETL (Extracción, Transformación y Carga) que facilita la preparación y carga de datos para análisis. Permite crear trabajos que transforman datos de diversas fuentes y los mueven a destinos como Amazon S3, Redshift o bases de datos.
      • Data Catalog: Actúa como un repositorio centralizado para la metadata de los datos. Permite a los usuarios descubrir y comprender los datos disponibles en la organización, facilitando el acceso a la información correcta para la toma de decisiones. El Data Catalog también se integra con AWS Glue para gestionar la metadata automáticamente.
      • Crawler: Los crawlers de AWS Glue son herramientas que escanean las fuentes de datos para identificar y catalogar la estructura y el formato de los datos. Esto automatiza el proceso de descubrimiento de datos, añadiendo automáticamente la metadata al Data Catalog y manteniéndolo actualizado.
    4. Analytics: El proceso de análisis y visualización de los datos previamente catalogados, normalizados, transformados y limpiados de manera que cumplan con la calidad de los datos esperados. En AWS implica utilizar servicios como Amazon Redshift y Amazon SageMaker para extraer, procesar y analizar datos de manera eficiente y escalable.
      • Amazon Redshift: Es un servicio de almacenamiento de datos en la nube que permite realizar consultas analíticas rápidas sobre grandes volúmenes de datos. Redshift utiliza una arquitectura de base de datos columnar y es optimizado para el análisis de datos, lo que permite a las organizaciones ejecutar consultas complejas y obtener informes detallados en tiempo real. Su capacidad de escalar y gestionar grandes conjuntos de datos lo hace ideal para la analítica empresarial.
      • Amazon SageMaker: Es un servicio completamente gestionado que facilita el desarrollo, entrenamiento y despliegue de modelos de machine learning. SageMaker permite a los usuarios construir modelos a partir de datos almacenados en Redshift, integrar diversas fuentes de datos y utilizar algoritmos predefinidos o personalizados. Esto permite a las organizaciones aplicar análisis predictivo y machine learning a sus datos para obtener insights más profundos y tomar decisiones informadas.

    Teniendo presente lo anterior de igual manera se puede ampliar mucho más las 3 primeras fases teniendo en cuenta que la relación entre la extracción y el tratamiento de los datos es muy importante, por esto, se plantean distintos flujos para llevar a cabo el tratamiento y administración de la data de la mejor manera y de la manera más rápida posible tal como se evidencia en la siguiente arquitectura.
